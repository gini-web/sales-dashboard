Google Apps Script 方案（規劃中，尚未實作）
  • 動機：目前的做法是「Claude 讀取 Sheet → 人工轉譯 → 寫成靜態網頁」，轉譯這一關容易出錯（例如經銷商名稱打錯字、漏抓超連結）。改用 Apps Script 可以讓 Sheet 與網頁之間沒有人工轉譯步驟，資料來源改什麼、網頁就顯示什麼，也能做到多人即時共用查看。
  • 架構：Google Sheet →（Apps Script 即時讀取＋運算分類）→ 部署成 Web App 固定網址 → 團隊隨時打開都是最新資料。
  • 如何自動抓「本週」分頁：分頁命名格式固定為「週報表M.D」；Script 會解析所有分頁名稱中的日期，自動選日期最新的一個當作本週資料（不依賴分頁順序）。若解析失敗，退回抓「最左邊的分頁」當保險。
  • 跨年判斷：因分頁名稱不含年份，若解析出的日期比今天晚超過約 2 週，會自動判定為「去年」而非「今年」，避免 12 月底／1 月初交界時判斷錯年份。
  • 部署方式：需要你自己到 Google Sheet「擴充功能→Apps Script」貼上程式碼，並「部署→新增部署作業→網頁應用程式」設定存取權限；之後我調整功能時，一樣需要你重新貼上並部署一次才會生效。
  • 維持不變的操作習慣：業務仍舊每週新增一個「週報表M.D」分頁、複製上週內容修改即可，不需要額外通知或改變填寫方式。
  • 狀態：目前尚未開始撰寫程式碼，這裡先記錄討論結論，之後决定要做時可以直接依此規格繼續。
